✨ oh-my-pi 是什么？

基于 TypeScript (Bun) + Rust 的高性能终端 AI 编码代理

oh-my-pi（命令行工具名 omp）是一款面向终端、功能极度完善的 AI 编码助手，由 Can Bölük 在 Mario Zechner 的 Pi 基础上 fork 并深度扩展而来。
它把 IDE 级别的能力带进了终端：LSP 智能代码导航、内置调试器、结构化子 Agent、代码审查、冲突解决，以及 32+ 内置工具，开箱即用。
同时 omp 支持 50+ 模型提供商（Anthropic、OpenAI、Google、DeepSeek、Groq、Mistral 等），并提供统一接口与高性能本地引擎（Rust grep、ast-grep、tree-sitter）。

💡 核心理念：一个 TUI 界面融合文件读写、代码执行、LSP、MCP、子代理并发，让终端成为最强大的 AI 开发环境。

📦 安装 omp

支持 macOS / Linux / Windows (Git Bash)

推荐：使用一键安装脚本（自动检测环境，配置 bash 路径）

# macOS / Linux curl -fsSL https://omp.sh/install | sh # Windows (PowerShell) irm https://omp.sh/install.ps1 | iex

备选：使用 Bun 全局安装（需先安装 Bun ≥ 1.3.14）

bun install -g @oh-my-pi/pi-coding-agent
🔔 Windows 用户注意：omp 需要 bash 环境（如 Git Bash）。安装器会尝试自动查找 bash.exe 并配置 shellPath。若失败，请手动在 ~/.omp/agent/settings.json 中指定。

⚙️ 初始配置 & API 认证

第一次运行 omp 时自动引导

任意项目目录执行 omp，进入 TUI 快速设置：

  • 选择模型提供商（Anthropic / OpenAI / Google / DeepSeek 等）
  • 输入 API Key（安全存储在 SQLite 数据库）
  • （可选）配置 OAuth 流程或云凭证（GCP ADC、AWS IAM）

配置目录遵循 XDG 规范，默认在 ~/.omp/：

文件/目录用途
~/.omp/agent/config.yml用户全局设置
.omp/settings.yml项目级配置（覆盖用户级）
~/.omp/agent/models.yml自定义模型与提供商定义
auth-storage.sqliteAPI Key / OAuth token 存储

除了 TUI 内配置，也支持环境变量：OPENAI_API_KEY、ANTHROPIC_API_KEY、DEEPSEEK_API_KEY 等，优先级依次为 运行时参数 > 项目配置 > 用户配置 > 默认值。

🚀 基础使用

交互式 TUI 与打印模式（Print Mode）

在项目中启动 omp，即进入交互式 TUI：

omp

也可以直接传递初始消息或附加文件：

omp "解释当前 React 组件的状态管理" --attach src/App.tsx

打印模式（Print Mode）：适合脚本、CI 集成，一次执行后退出。

omp -p "用一句话解释 Docker" --mode text omp --mode json -p '{"prompt":"列出根目录文件"}'

TUI 核心能力：实时流式输出、多行编辑器、Markdown/图片渲染、斜杠命令、原生 Bash/Python 执行、会话分支与上下文压缩。

⌨️ 内置斜杠命令（Slash Commands）

在对话框中输入 / 唤起命令，不向 LLM 发送请求，直接控制会话

命令语法示例说明
/settings/settings打开设置菜单（主题、模型、工具开关等）
/plan/plan 或 /plan 实现用户登录API开启计划模式（先规划后执行）
/model/model切换当前模型（如切换 DeepSeek-V4 / Claude 3.5）
/fast/fast on切换 OpenAI 服务层优先级
/session/session打开会话列表（切换/删除/分支）
/export/export ~/chat.html将会话导出为 HTML 文件
/goal/goal set "修复所有测试"持久化自治任务目标
/loop/loop 5自动循环提交 yield，用于无人值守
/compact/compact手动触发上下文压缩
/help/help显示帮助与快速参考
💡 扩展斜杠命令：你可以在 ~/.omp/agent/commands/[name]/index.ts 中编写 TypeScript 自定义命令，并使用 ExtensionAPI 调用核心能力。

🖥️ 命令行选项（CLI Flags）

灵活控制模型、会话与输出模式

参数简写描述
--model / --provider指定模型或提供商，如 --model claude-3-5-sonnet
--smol / --slow / --plan为不同角色分配模型（快速/慢速/规划）
--thinking开启推理模式 low|medium|high|xhigh
--resume-r恢复会话（交互选择或指定ID）
--continue-c恢复当前目录最近会话
--fork基于现有会话创建新分支
--no-session无持久化内存模式
--modetext|json|rpc|acp，切换打印模式或 RPC 模式
# 使用规划模型，开启高程度推理 omp --plan "refactor auth module" --thinking high

🧠 高级特性

Plan Mode · Subagents · MCP · 自治记忆 · 后台作业

📐 Plan Mode（计划模式）
输入 /plan 后进入只读架构阶段，Agent 生成 PLAN.md 后再执行；防止 AI 意外修改文件，适合复杂重构。
✨ 提供三种批准方式：直接执行 / 压缩上下文 / 保留当前历史。
🐝 任务委托与 Subagents
使用 task 工具可以生成多个隔离子代理并行工作，支持扇出工作流。底层通过 Semaphore 控制并发。
子代理可运行在同一进程或独立子进程中。
🔌 MCP 集成
支持 stdio / HTTP / SSE 传输，自动发现外部 MCP 服务器工具，并桥接到 Agent 工具集。支持 OAuth token 刷新与发现模式 (search_tool_bm25)。
🧠 自治记忆 (Hindsight Memory)
从历史会话中提取知识并构建“心智模型”，实现长期记忆，减少重复解释项目结构。
🔗 后台作业：Bash 命令和子代理任务可以放入后台，不阻塞主推理循环。使用 /jobs 查看进度。

🔧 配置参考

分层配置、模型路由与环境变量

配置优先级（高→低）： CLI 参数 > 项目 .omp/settings.yml > 用户 ~/.omp/agent/config.yml > 默认值。

models.yml 可以自定义模型别名、上下文窗口、token 价格。例如接入 DeepSeek：

providers:
  - name: deepseek
    apiBase: https://api.deepseek.com/v1
    apiKey: ${DEEPSEEK_API_KEY}
models:
  - id: deepseek-chat
    name: DeepSeek-V4
    provider: deepseek

环境变量补充：OPENAI_API_KEY、ANTHROPIC_API_KEY、PI_NATIVE_VARIANT 等。系统自动加载 project.env、.env。

💡 交互式配置：TUI 中输入 /settings 可实时修改主题、颜色盲模式、搜索提供商（Tavily/Perplexity/Brave）、状态栏布局。

❓ 常见问题 & 调试

  • Q：Windows 下 omp 无法启动或提示 bash 未找到？
    安装器会自动查找 Git Bash 路径，也可手动修改 ~/.omp/agent/settings.json 中的 shellPath 为正确的 bash.exe。
  • Q：如何查看当前会话 ID / 历史？
    使用 /session 命令，支持切换、删除、分支会话。会话文件默认保存在 ~/.omp/agent/sessions/。
  • Q：某些 MCP 工具未加载？
    检查 settings.yml 中 mcpServers 配置，确保命令与传输类型正确；使用 --debug 启动 omp 查看详细日志。
  • Q：编辑冲突或哈希行更新失败？
    omp 默认使用 hashline 编辑模式，基于代码锚点更新，如果文件发生大幅变化，可以临时切换编辑模式或在 /settings 中调整 editMode。
  • Q：模型 API key 验证失败？
    确认环境变量或 models.yml 中的 apiKey 正确。对于 DeepSeek，目前需手动设置 DEEPSEEK_API_KEY，暂未内置 OAuth。
  • Q：omp 与 Claude Code 或 Cursor 的关系？
    omp 是独立的高性能终端代理，可以读取 .cursor/rules 或 .claude/settings.yml 实现零迁移成本，但同时也内置 LSP/Debugger 等 IDE 能力，定位不同。
🐛 调试技巧：omp --debug 启动会打印详细工具调用链、MCP 连接、子代理输出，便于排查性能或兼容问题。
