MiMo-Code 使用说明

小米 MiMo 团队开源的终端 AI 编程 Agent，具备跨会话记忆、智能上下文管理和自主进化能力

MIT License 开源 MiMo Auto 限时免费
快速开始 核心特性 工作模式 斜杠命令 内置工具 快捷键 配置 记忆系统 最佳实践 故障排除

🚀 快速开始

前提条件

  • 现代终端模拟器（WezTerm / Alacritty / Ghostty / Kitty / Windows Terminal）
  • LLM 提供商的 API 密钥（或使用免费的 MiMo Auto）

安装

Mac / Linux（推荐）：

curl -fsSL https://mimo.xiaomi.com/install | bash

Windows / npm：

npm install -g @mimo-ai/cli

Nix：

nix run github:XiaomiMiMo/MiMo-Code

首次启动

在项目目录中运行 mimocode，首次启动会引导你选择模型接入方式：

通过 OAuth 登录小米 MiMo 平台

添加任何 OpenAI 兼容的 API

✨ 核心特性

🧠 跨会话记忆

基于 SQLite FTS5 全文搜索的持久化记忆系统。项目记忆、会话检查点、任务进度在重启后自动恢复，无需重新学习项目上下文。

📐 智能上下文管理

自动检查点保存、上下文重建和预算注入。当上下文接近上限时，从最新的检查点和记忆文件中恢复状态，逻辑会话可以无限延伸。

🔄 任务追踪

树形任务系统（T1, T1.1, T1.2...）自动与检查点集成，会话恢复时任务进度完整保留。

🤖 子代理系统

主代理可以按需创建子代理，并行工作，支持生命周期追踪、取消和后台执行。

🎯 Goal 完成度验证

设定停止条件后，独立审查模型验证是否真正完成，防止提前终止或过度运行。

🧪 Max Mode 并行采样

每轮并行生成 N 个候选方案（默认 5），由 judge 模型选出最优执行，SWE-Bench Pro 提升 10-20%。

🗣️ 语音输入

实时流式语音输入，通过 /voice 激活，支持增量转录。

💤 Dream & Distill

Dream 每 7 天自动整理记忆；Distill 每 30 天发现重复工作流并固化为可复用的 skill 或命令。

⚙️ 配置

配置文件位置：.mimocode/mimocode.json（项目级）或 ~/.config/mimocode/mimocode.json（全局）。

完整配置示例

{
  "$schema": "https://mimo.xiaomi.com//config.json",
  "model": "mimo/mimo-v2.5-pro",
  "small_model": "mimo/mimo-v2.5-pro",
  "default_agent": "build",
  "share": "manual",
  "autoupdate": true,

  "provider": {
    "anthropic": {
      "options": { "timeout": 600000, "setCacheKey": true }
    }
  },

  "permission": {
    "*": "ask",
    "bash": "allow",
    "edit": "deny"
  },

  "compaction": {
    "auto": true,
    "prune": true,
    "tail_turns": 2
  },

  "watcher": {
    "ignore": ["node_modules/**", "dist/**"]
  },

  "instructions": ["CONTRIBUTING.md"],

  "experimental": {
    "maxMode": false
  }
}

关键字段说明

字段说明
model默认模型，格式 provider_id/model_id
default_agent默认代理（build / plan / compose）
permission工具权限（allow / ask / deny），支持通配符
compaction上下文压缩策略（auto / prune / tail_turns）
instructions注入额外指令文件路径或 glob
sharemanual / auto / disabled
autoupdatetrue / false / "notify"
experimental.maxMode启用并行采样选优（实验性）
💡 变量替换 配置值支持 {env:VAR} 和 {file:path} 占位符，方便引用环境变量和密钥文件。

🧠 记忆系统

MiMo Code 采用四层记忆架构，越上层越精炼持久，越下层越完整庞大。

Session 记忆

checkpoint.md — 当前逻辑会话的完整工作状态，由独立 writer subagent 自动维护。11 个结构化字段：意图、下一步、任务树、涉及文件、错误等。

Project 记忆

MEMORY.md — 项目级持久知识：架构决定、用户规则、技术事实。跨 session 稳定的内容从 session 层提升至此。

Global 记忆

用户级偏好，跨项目生效。

History

SQLite 完整轨迹存储，每条消息和工具调用原文。当结构化记忆找不到细节时向下回溯。

检查点机制

  • 在已配置预算的 20%、45%、70% 处触发检查点
  • 每次触发都是对前一次的增量更新
  • Writer subagent 独立于主 Agent 执行，互不干扰
  • 重建时将持久化文件组装为分层 prompt 注入新窗口（约 65K token 以内）
📝 notes.md — 唯一的写入通道 主 Agent 只能读取结构化文件，但可以随时 append 到 notes.md。Writer 在每次 checkpoint 时读取、路由到对应字段后清空。

🏆 最佳实践

1. 善用 Plan 模式先规划再执行

对于复杂任务，先切换到 plan 模式（Tab），让 AI 分析代码库并制定方案。确认后再切回 build 执行。这可以避免不必要的文件修改。

2. 利用 Goal 防止提前终止

对于自动化任务，设置明确的 Goal 停止条件（如"所有测试通过且代码已提交"），让独立验证模型确保任务真正完成。

3. 使用 AGENTS.md 定义项目规则

运行 /init 生成 AGENTS.md，在其中定义项目特定的编码规范、架构约束和偏好。这些规则会在每次会话中自动注入。

4. 合理配置权限

建议将 bash 和 edit 设为 "ask"，让 AI 在执行危险操作前确认。对于信任的项目可以设为 "allow"。

{
  "permission": {
    "*": "allow",
    "bash": "ask",
    "edit": "ask"
  }
}

5. 利用 Compose 模式进行 TDD

切换到 compose 模式后，使用 compose:tdd 技能进行测试驱动开发。先写测试，再实现功能，自动验证。

6. 定期整理记忆

Dream（每 7 天自动触发）和 Distill（每 30 天自动触发）会自动维护记忆质量。你也可以手动运行 /dream 整理记忆。

7. 使用 Undo/Redo 安全回滚

/undo 会撤销最后一条消息及所有文件变更（基于 Git），/redo 重做。确保项目是 Git 仓库即可使用。

8. 利用 Subagent 并行处理

对于大型任务，子代理可以并行处理不同模块。主代理负责协调，子代理独立执行，提高整体效率。

9. 启用 Max Mode 提升准确率

对于关键任务，在配置中启用 "experimental": { "maxMode": true }，并行生成 5 个候选方案由 judge 选出最优。

10. 利用 MCP 扩展能力

通过 MCP 服务器集成外部工具（数据库、API、第三方服务），将 MiMo Code 的能力扩展到代码编辑之外。

🛠️ 故障排除

Shift+Enter 无法换行

某些终端默认不发送修饰键 Enter。Windows Terminal 需在 settings.json 中添加：

"actions": [
  {
    "command": { "action": "sendInput", "input": "\u001b[13;2u" },
    "id": "User.sendInput.ShiftEnterCustom"
  }
]
"keybindings": [
  { "keys": "shift+enter", "id": "User.sendInput.ShiftEnterCustom" }
]

搜索不到被 .gitignore 排除的文件

在项目根目录创建 .ignore 文件，添加允许的路径：

!node_modules/
!dist/
!build/

undo/redo 不工作

确保项目是一个 Git 仓库。undo/redo 通过 Git 管理文件变更。

websearch 不可用

需要使用 MiMo 提供商，或设置环境变量 MIMOCODE_ENABLE_EXA=1。

⚠️ LSP 工具为实验性功能 需要设置 MIMOCODE_EXPERIMENTAL_LSP_TOOL=true 或 MIMOCODE_EXPERIMENTAL=true 才能启用。
